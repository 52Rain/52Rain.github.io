2.3 添加密钥

  1. 进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”）
  2. 用记事本打开公钥 id_rsa.pub 文件并复制里面的内容
  3. 登陆 GitHub ，进入 Settings 页面（点击头像进入）
  4. 选择左边栏的 SSH and GPG keys，点击 New SSH key
  5. Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加



2.4 验证连接

打开 Git Bash，输入 ssh -T git@github.com 出现 “Are you sure……”，输入 yes 回车确认。

显示 “Hi xxx! You’ve successfully……” 即连接成功。

三、创建 Github Pages 仓库

  1. GitHub 主页右上角加号 -> New repository
  2. Repository name 中输入 用户名.github.io
  3. 勾选 “Add a README file”
  4. Description 选填
  5. 填好后点击 Create repository 创建。
  6. 创建后默认自动启用 HTTPS，博客地址为：https://用户名.github.io

*可能遇到的问题

a.不知是否部署成功

查看之前创建的库里有没有文件，如果没有文件或只有一个.md文件，则没有部署成功

b.怎么修改默认分支

进入储存库的设置界面
点击交换符号
点击下拉选择按钮，选择分支，并点击update更新，在出现的弹窗点击确认即可

c.Hexo部署出现错误

报错如下：err: Error: Spawn failed
参考教程：https://blog.csdn.net/weixin_41256398/article/details/117994899

d.部署成功后，页面只显示网站没有内容

检查 _config.yml文件的分支与GitHub库的分支是否对应，检查 _config.yml文件的url是否正确，若仍无法解决问题，请查看官方文档–>部署
本地预览正常，确定部署上去后，需强制刷新浏览器缓存，Shift+F5刷新，才能显示修改后的新界面

e.命令行输入hexo d后发现不能输入GitHub密码登录

参考教程：https://blog.csdn.net/qq_43340547/article/details/119895671
